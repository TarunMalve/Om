# Architectural Decision Record (ADR)
## ADR-001: Hybrid Ontology Architecture (RDF/OWL & LPG)

### Context & Problem Statement
The platform must scale to 1M+ nodes and 100M+ users. We require strict semantic validation, academic standard interoperability, and high-performance, low-latency graph traversals (real-time recommendation and GraphRAG). No single graph system (pure RDF/Triple Store or pure LPG/Neo4j) satisfies both.

### Decision
We will employ a **Hybrid Ontology Architecture**:
1. Standardize core schema definitions using RDF/OWL to leverage academic tools, ontology schemas, and SHACL shapes validation.
2. Ingest RDF triples directly into Neo4j Labeled Property Graphs (LPG), converting classes to node labels, object properties to directed edge labels, and datatype properties to key-value properties.

### Consequences
* **Pros**: 
  - Dual-mode queries (SPARQL and Cypher).
  - Strict validation of schemas via SHACL shapes.
  - Low-latency real-time traversals for AI context.
* **Cons**:
  - Requires maintaining a pipeline mapping RDF schemas to Neo4j.
